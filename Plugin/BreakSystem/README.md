# ブレイクシステム

戦略的な駆け引きを生み出す「ブレイクシステム」を実装するプラグインです。  
相性有利な武器での攻撃が命中した際に、相手ユニットを「ブレイク状態」にして反撃を封じることができます。

[![Version](https://img.shields.io/badge/version-1.33-blue.svg)]()
[![SRPG Studio](https://img.shields.io/badge/SRPG%20Studio-1.313-green.svg)]()

## 📋 機能

### ブレイク状態の効果

<!-- TODO:ここに戦闘予測画面のスクショ（ブレイク状態で攻撃・命中・必殺が空欄になっている様子） -->

-   ✅ **反撃無効化**: 現在の戦闘および次の戦闘まで、一切の反撃が不可能
-   ✅ **自動回復**: 所属勢力のフェイズ開始時に自動的に解除
-   ✅ **UI 表示**: 戦闘予測画面で攻撃・命中・必殺が空欄表示

### 発動条件

1. 戦闘を仕掛けた側のユニットが攻撃
2. 相性有利な武器での攻撃
3. 攻撃が命中しダメージを与える
4. 相手が「ブレイク無効」スキルを持たない

### 敵 AI の思考ルーチンの変更

-   敵 AI がブレイク可能な相手を優先的に狙う設定が可能
-   優先度は設定で調整可能（デフォルト：1.5 倍）

## 🚀 導入方法

`BreakSystem.js`を SRPG Studio のプロジェクトの Plugin フォルダに配置する

## ⚙️ 基本設定

### 1. ブレイク状態の作成

1. **ステートを新規作成**し、以下の設定を行う：

<!-- TODO:ステート設定画面のスクショ -->

| 項目                 | 設定値                                   |
| -------------------- | ---------------------------------------- |
| 名前                 | 任意（例：「ブレイク」）                 |
| マップアニメーション | 任意                                     |
| リアルアニメーション | 任意                                     |
| 持続ターン           | **1**                                    |
| 自動解除条件         | **「戦闘に入った」かつ「1 回目で解除」** |
| その他               | デフォルト値のまま                       |

### 2. プラグイン設定

`BreakSystem.js`の以下の設定項目を編集：

```javascript
// ブレイク状態のステートのID
var BREAK_STATE_ID = 6; // ← 作成したステートのIDに変更
```

## 🎮 カスタマイズ

### 設定項目一覧

| 設定項目                                | デフォルト値 | 説明                                 |
| --------------------------------------- | ------------ | ------------------------------------ |
| `WAIT_TURN_SYSTEM_COEXISTS`             | `false`      | ウェイトターンシステムとの併用設定   |
| `IS_SPECIFIED_SKILL_OR_WEAPON_REQUIRED` | `false`      | 特定スキル/武器でのみブレイク可能    |
| `BREAK_STATE_ID`                        | `6`          | ブレイク状態のステート ID            |
| `BREAK_PRIORITY_RATE`                   | `1.5`        | 敵 AI がブレイク可能相手を狙う優先度 |
| `BREAK_WEAPON_SENTENCE_TEXT`            | `"ブレイク"` | 武器情報ウィンドウの表示文字列       |

### 高度な設定

#### ブレイク無効スキル

特定のユニット（ボス等）をブレイク不可にする場合：

1. カスタムスキルを作成
2. キーワードに `resistBreak` を設定
3. 対象ユニットにスキルを付与

#### ブレイク専用スキル・武器

特定の条件でのみブレイクを発動させる場合：

1. `IS_SPECIFIED_SKILL_OR_WEAPON_REQUIRED` を `true` に設定

2. **ブレイクスキル**を作成：

    ```javascript
    // カスタムスキルのキーワードに「break」を設定
    ```

3. **ブレイク武器**を設定：
    ```javascript
    // 武器のカスタムパラメータ
    {
        isBreakWeapon: true;
    }
    ```

## 💡 使用例

### 基本的な戦術

<!-- TODO:槍兵が剣兵をブレイクし、その後に別のユニットが安全に追撃しているgif -->

```
1. 槍兵が剣兵を攻撃（相性有利：槍 > 剣）
2. 攻撃が命中してダメージを与える
3. 剣兵が「ブレイク状態」になり反撃不可
4. 他のユニットで安全に追撃可能
```

### 戦略的な活用

-   **強敵ボス攻略**: 反撃を恐れずに攻撃できる
-   **連携攻撃**: ブレイク後に複数ユニットで追撃
-   **相性重視**: 武器相性の戦略的重要性が向上

## 🔗 他プラグインとの連携

### ウェイトターンシステム

併用する場合の設定：

```javascript
var WAIT_TURN_SYSTEM_COEXISTS = true; // falseからtrueに変更
```

**動作の違い**：

-   通常：所属フェイズ開始時にブレイク状態解除
-   ウェイトターン併用時：対象ユニットのアタックターン開始時に解除

### 互換性

-   ✅ SRPG Studio version 1.313 で動作確認済み

## 🔧 トラブルシューティング

### よくある問題

**Q: ブレイク状態が付与されない**

```
A: 以下を確認してください：
✓ ステートIDが正しく設定されているか
✓ 相性有利な攻撃になっているか
✓ ダメージが発生しているか（ガード等でダメージ0の場合は付与されない）
✓ 相手が「resistBreak」スキルを持っていないか
```

**Q: ブレイク状態が解除されない**

```
A: ステートの自動解除設定を確認してください：
✓ 持続ターンが「1」になっているか
✓ 自動解除条件が「戦闘に入った」「1回目で解除」になっているか
```

**Q: 敵 AI がブレイクを狙わない**

```
A: 優先度設定を確認してください：
✓ BREAK_PRIORITY_RATEが1より大きい値になっているか
✓ 相性有利な攻撃が可能な状況になっているか
```

### デバッグ情報

設定確認用のコード：

```javascript
// コンソールで設定値を確認
root.log("BREAK_STATE_ID:", BREAK_STATE_ID);
root.log("BREAK_PRIORITY_RATE:", BREAK_PRIORITY_RATE);
```

## 📖 概要

BreakSystem プラグインは、相性有利な武器での攻撃が命中した際に、相手ユニットを「ブレイク状態」にして反撃を封じるシステムです。武器相性の戦略的重要性を高め、より深い戦術を可能にします。

### 主な特徴

-   ✅ **反撃無効化** - ブレイク状態では一切の反撃が不可能
-   ✅ **自動回復** - 所属勢力のフェイズ開始時に自動的に解除
-   ✅ **AI 対応** - 敵 AI がブレイク可能な相手を優先的に狙う
-   ✅ **カスタマイズ可能** - 特定スキルや武器でのみ発動する設定が可能
-   ✅ **ブレイク無効スキル** - 特定ユニットをブレイク不可に設定可能
-   ✅ **他プラグイン連携** - ウェイトターンシステムとの併用対応

## ⚙️ 機能

### ブレイク状態の効果

-   ✅ **反撃無効化**: 現在の戦闘および次の戦闘まで、一切の反撃が不可能
-   ✅ **自動回復**: 所属勢力のフェイズ開始時に自動的に解除
-   ✅ **UI 表示**: 戦闘予測画面で攻撃・命中・必殺が空欄表示

### 発動条件

1. 戦闘を仕掛けた側のユニットが攻撃
2. 相性有利な武器での攻撃
3. 攻撃が命中しダメージを与える
4. 相手が「ブレイク無効」スキルを持たない

### 敵 AI の思考ルーチンの変更

-   敵 AI がブレイク可能な相手を優先的に狙う設定が可能
-   優先度は設定で調整可能（デフォルト：1.5 倍）

## 🚀 インストール

### 手順 1: ファイルの配置

1. `BreakSystem.js`をプロジェクトの`Plugin`フォルダに配置
2. SRPG Studio でプロジェクトを開く
3. プラグインが自動的に読み込まれます

### 手順 2: 動作確認

プロジェクトを実行し、エラーが発生しないことを確認してください。

## 🔧 設定方法

### 1. ブレイク状態の作成

1. **ステートを新規作成**し、以下の設定を行う：

| 項目                 | 設定値                                   |
| -------------------- | ---------------------------------------- |
| 名前                 | 任意（例：「ブレイク」）                 |
| マップアニメーション | 任意                                     |
| リアルアニメーション | 任意                                     |
| 持続ターン           | **1**                                    |
| 自動解除条件         | **「戦闘に入った」かつ「1 回目で解除」** |
| その他               | デフォルト値のまま                       |

### 2. プラグイン設定

`BreakSystem.js`の以下の設定項目を編集：

```javascript
// ブレイク状態のステートのID
var BREAK_STATE_ID = 6; // ← 作成したステートのIDに変更
```

**効果**: 特定のスキルを持つユニットのみブレイク可能

#### スキル設定

2. キーワードに `break` を設定
3. 対象ユニットにスキルを付与

### 例 3: ウェイトターンシステム連携

**効果**: ウェイトターンシステムとの併用

#### プラグイン設定

```javascript
var WAIT_TURN_SYSTEM_COEXISTS = true;
```

**効果の違い**:

-   通常: 所属フェイズ開始時にブレイク状態解除
-   ウェイトターン併用時: 対象ユニットのアタックターン開始時に解除

## 🎮 高度な使い方

### カスタマイズ設定

| 設定項目                                | デフォルト値 | 説明                                 |
| --------------------------------------- | ------------ | ------------------------------------ |
| `WAIT_TURN_SYSTEM_COEXISTS`             | `false`      | ウェイトターンシステムとの併用設定   |
| `IS_SPECIFIED_SKILL_OR_WEAPON_REQUIRED` | `false`      | 特定スキル/武器でのみブレイク可能    |

| `BREAK_WEAPON_SENTENCE_TEXT`            | `"ブレイク"` | 武器情報ウィンドウの表示文字列       |

### 高度な設定

#### ブレイク無効スキル

特定のユニット（ボス等）をブレイク不可にする場合：

1. カスタムスキルを作成
2. キーワードに `resistBreak` を設定
3. 対象ユニットにスキルを付与

#### ブレイク専用スキル・武器

特定の条件でのみブレイクを発動させる場合：

1. `IS_SPECIFIED_SKILL_OR_WEAPON_REQUIRED` を `true` に設定

2. **ブレイクスキル**を作成：

    ```javascript
    // カスタムスキルのキーワードに「break」を設定
    ```

3. **ブレイク武器**を設定：
    ```javascript
    // 武器のカスタムパラメータ
    {
        isBreakWeapon: true;
    }
    ```

## 💡 実用例

### 基本的な戦術

```
1. 槍兵が剣兵を攻撃（相性有利：槍 > 剣）
2. 攻撃が命中してダメージを与える
3. 剣兵が「ブレイク状態」になり反撃不可
4. 他のユニットで安全に追撃可能
```

### 戦略的な活用

-   **強敵ボス攻略**: 反撃を恐れずに攻撃できる
-   **連携攻撃**: ブレイク後に複数ユニットで追撃
-   **相性重視**: 武器相性の戦略的重要性が向上
-   **AI 戦術**: 敵 AI も相性有利な攻撃を積極的に狙ってくる

### シナリオ別使用例

#### シナリオ 1: 強敵ボス戦

-   **状況**: HP・攻撃力が高く、反撃が危険なボス
-   **戦術**: 相性有利な武器でブレイクを狙い、無反撃状態で集中攻撃
-   **効果**: 安全にダメージを与え続けることができる

#### シナリオ 2: 大軍との戦い

-   **状況**: 多数の敵に囲まれた危険な状況
-   **戦術**: 前衛をブレイクして反撃を封じ、後続を処理
-   **効果**: 被ダメージを大幅に削減して安全に戦線を突破

#### シナリオ 3: 相性不利な敵との戦い

-   **状況**: こちらが相性不利で正面から戦えない
-   **戦術**: 相性有利なユニットでブレイクを狙い、相性不利なユニットで追撃
-   **効果**: 相性の壁を越えて効率的にダメージを与える

## 🔧 トラブルシューティング

### よくある問題

**Q: ブレイク状態が付与されない**

```
A: 以下を確認してください：
✓ ステートIDが正しく設定されているか
✓ 相性有利な攻撃になっているか
✓ ダメージが発生しているか（ガード等でダメージ0の場合は付与されない）
✓ 相手が「resistBreak」スキルを持っていないか
```

**Q: ブレイク状態が解除されない**

```
A: ステートの自動解除設定を確認してください：
✓ 持続ターンが「1」になっているか
✓ 自動解除条件が「戦闘に入った」「1回目で解除」になっているか
```

**Q: 敵 AI がブレイクを狙わない**

```
A: 優先度設定を確認してください：
✓ BREAK_PRIORITY_RATEが1より大きい値になっているか
✓ 相性有利な攻撃が可能な状況になっているか
```

### デバッグ情報

設定確認用のコード：

```javascript
// コンソールで設定値を確認
root.log("BREAK_STATE_ID:", BREAK_STATE_ID);
root.log("BREAK_PRIORITY_RATE:", BREAK_PRIORITY_RATE);
```

## 🔌 API 仕様

### 公開関数

本プラグインは現在外部公開 API を持ちませんが、以下の内部関数が主要な役割を担っています。

#### 内部主要関数

-   `_canBreak(active, passive)`: ブレイク可能かどうかを判定
-   `_hasBreakSkill(unit)`: ユニットがブレイクスキルを持つかを判定
-   `_hasBreakWeapon(weapon)`: 武器がブレイク武器かを判定
-   `_hasResistBreakSkill(unit)`: ユニットがブレイク無効スキルを持つかを判定

### 拡張ポイント

    if (state.getId() === BREAK_STATE_ID && increaseType === IncreaseType.INCREASE) {
        // カスタム処理をここに追加
    }

    return result;
};

### パフォーマンス最適化

本プラグインは Ver.1.31 で以下の最適化を実施しています：

-   **重複コードの削除**: ステート取得、配列フィルタリングの共通化
-   **関数分割**: 長い関数を単一責任原則に基づいて分割
-   **ヘルパー関数**: 共通処理の抽出によるコードの再利用性向上

### コード品質

-   **JSDoc コメント**: 全関数に型情報と説明を記載
-   **単一責任原則**: 各関数が明確な責務を持つよう設計
-   **命名規則**: `_original_`プレフィックスで元の関数を明確化

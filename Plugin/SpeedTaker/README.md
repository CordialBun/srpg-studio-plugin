## 📖 概要

SpeedTaker プラグインは、ユニットが自分から戦闘を仕掛けて相手を撃破した際に、指定した能力値が上昇するバフステートを獲得できるようになるプラグインです。効果は累積し、マップ終了まで継続します。

### 主な特徴

-   ✅ **柔軟な能力値設定** - 速さ以外の全能力値に対応
-   ✅ **累積効果** - 撃破数に応じて効果が累積（上限設定可能）
-   ✅ **視覚的フィードバック** - ステートアイコン上に現在の上昇値を表示
-   ✅ **全勢力対応** - 自軍・敵軍・同盟軍のユニットが使用可能
-   ✅ **複数スキル同時保有** - 異なる能力値を対象とする複数の吸収スキルを同時に使用可能
-   ✅ **デバフ対応** - マイナス値を設定することでデバフスキルとしても利用可能

## ⚙️ 機能

### 基本動作

1. **発動条件**
    - 自分から戦闘を仕掛ける
    - 相手ユニットを撃破する
2. **効果**

    - 指定した能力値が設定値分上昇
    - 効果は累積（上限まで）
    - マップ終了まで継続

3. **表示**
    - ステートアイコンで状態を表示
    - アイコン上に現在の上昇値を数値表示

## 🚀 インストール

### 手順 1: ファイルの配置

1. `SpeedTaker.js`をプロジェクトの`Plugin`フォルダに配置
2. SRPG Studio でプロジェクトを開く
3. プラグインが自動的に読み込まれます

### 手順 2: 動作確認

プロジェクトを実行し、エラーが発生しないことを確認してください。

## 🔧 設定方法

### 手順 1: カスタムステートの作成

1. **データベースを開く**

    - SRPG Studio でプロジェクトを開く
    - `データ設定` → `ステート` を選択

2. **新規ステートを作成**

    - 「ステートの作成」ボタンをクリック
    - 名前を設定（例：「速さ吸収」）

3. **カスタムパラメータを設定**

    ```json
    {
        "isSpeedTakerState": true,
        "paramType": 4,
        "countLimit": 5,
        "perBonus": 2
    }
    ```

4. **その他の設定**
    - 「バッドステータスとして扱う」のチェックを**外す**
      ※バッドステータス回復アイテムや回復魔法で除去されないようにするため
    - アイコンを設定（任意）

### 手順 2: カスタムスキルの作成

1. **データベースを開く**

    - `データ設定` → `スキル` を選択

2. **新規スキルを作成**

    - 「スキルの作成」ボタンをクリック
    - 名前を設定（例：「速さの吸収」）

3. **キーワードを設定**

    - キーワード欄に `SpeedTaker` と入力（大文字小文字を正確に）

4. **カスタムパラメータを設定**
    ```json
    {
        "stateId": 15
    }
    ```
    ※ `stateId`の値は手順 1 で作成したステートの ID に置き換えてください

## 💡 実用例

### 基本的な戦術

```
1. 速さ吸収スキルを持つユニットで雑魚敵を撃破
2. 撃破数を稼いで能力値を強化
3. 強化された能力でボス戦に挑む
4. 高い速さで連続攻撃、高い回避率で生存性向上
```

### 戦略的な活用

-   **雑魚狩り特化**: 雑魚を積極的に狩って能力値を強化
-   **スノーボール戦術**: 少数の強いユニットに集中強化
-   **バランス型**: 異なる能力値を同時に吸収して万能型ユニットを育成
-   **デバフ戦術**: マイナス吸収で敵を弱体化しつつ自身は強化

### シナリオ別使用例

#### シナリオ 1: 速さ重視マップ

-   **目的**: 高速戦闘ユニットの育成
-   **設定**: 速さ+3、上限 5 回の吸収スキル
-   **効果**: 最終的に速さ+15 の超高速ユニットが完成

#### シナリオ 2: ボス戦特化

-   **目的**: 強力なボスに対するアタッカー育成
-   **設定**: 力+4、上限 8 回の吸収スキル
-   **効果**: 最終的に力+32 の破壊力を持つアタッカーが完成

## 🎮 高度な使い方

### 複数の吸収スキルを同時使用

同一ユニットに複数の吸収スキルを持たせることができます。

**例**: 速さと力を同時に吸収

1. 速さ吸収用のステートとスキルを作成
2. 力吸収用のステートとスキルを作成
3. 両方のスキルをユニットに付与

### 条件付き吸収

クラスやユニット固有のスキルとして設定することで、特定の条件下でのみ発動する吸収スキルを作成できます。

## 🔌 API 仕様

### 公開関数

#### SGP_drawCurTakenBonus

ステートアイコンの上に現在の能力値上昇量を表示する関数です。他の UI プラグインとの競合時に外部から呼び出すことができます。

```javascript
SGP_drawCurTakenBonus(x, y, turnState);
```

**パラメータ**:

-   `x` (number): 描画位置の X 座標
-   `y` (number): 描画位置の Y 座標
-   `turnState` (Object): ターンステートオブジェクト

**使用例**:

```javascript
// 他のプラグインから呼び出す場合
SGP_drawCurTakenBonus(100, 200, turnState);
```

## 🛠️ トラブルシューティング

### Q: スキルが発動しない

**A**: 以下を確認してください：

1. スキルのキーワードが正確に `SpeedTaker` になっているか
2. カスタムパラメータの JSON 形式が正しいか
3. ステート ID が正しく設定されているか
4. 「バッドステータスとして扱う」のチェックが外れているか

### Q: 数値が表示されない

**A**: ステートアイコンが設定されていることを確認してください。アイコンがないと数値も表示されません。

### Q: 他の UI プラグインと競合する

**A**: 以下の手順で対処してください：

1. 本プラグインの 466〜482 行目（`UnitSentence.State.drawUnitSentence`のオーバーライド部分）をコメントアウト

2. UI を変更しているプラグイン内で以下の記述を探す：

    ```javascript
    GraphicsRenderer.drawImage(x, y, state.getIconResourceHandle(), GraphicsType.ICON);
    ```

3. 見つかった場合、その直後に以下を追記：
    ```javascript
    SGP_drawCurTakenBonus(x, y, turnState);
    ```

### Q: 効果が累積しない

**A**: `countLimit`の値を確認してください。1 に設定されていると累積しません。

## ⚡ 技術仕様

### 設定項目一覧

本プラグインはコード内で以下の設定項目をカスタマイズできます。

| 設定項目               | デフォルト値   | 説明                               |
| ---------------------- | -------------- | ---------------------------------- |
| `SPEED_TAKER_KEYWORD`  | `"SpeedTaker"` | 吸収スキルを識別するキーワード     |
| `ICON_SPACING`         | `22`           | ステートアイコン間の水平スペース   |
| `ICONS_PER_ROW`        | `2`            | 1 行あたりのステートアイコン表示数 |
| `MIN_COUNT_LIMIT`      | `1`            | 累積回数の最小制限値               |
| `INITIAL_DEFEAT_COUNT` | `1`            | 初回撃破時の撃破カウント初期値     |

### パフォーマンス最適化

本プラグインは Ver.1.01 で以下の最適化を実施しています：

-   **早期判定**: SpeedTaker ステート以外を即座にスキップ
-   **インライン化**: 頻繁に呼ばれる関数呼び出しを削減
-   **型チェック最適化**: 複数の型チェックを一度に実行

### コード品質

-   **JSDoc コメント**: 全関数に型情報と説明を記載
-   **単一責任原則**: 長い関数を小さな関数に分割
-   **命名規則**: `_original_`プレフィックスで元の関数を明確化

### アーキテクチャ

```
├── 外部API関数
│   └── SGP_drawCurTakenBonus（他プラグインとの競合対応用公開関数）
├── 設定項目
│   └── 各種定数定義（キーワード、UI配置、制限値等）
├── ステート効果機能
│   └── StateControl.getStateParameter（能力値計算のオーバーライド）
├── ステート管理機能
│   ├── PreAttack._doEndAction（戦闘終了処理のオーバーライド）
│   └── 各種ヘルパー関数（型安全なアクセサ、検索、更新処理）
└── UI表示機能
    └── UnitSentence.State.drawUnitSentence（ステートアイコン描画のオーバーライド）
```
